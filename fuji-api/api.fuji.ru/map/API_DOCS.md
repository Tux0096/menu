# Документация API зон доставки

Данная документация описывает API для управления зонами доставки в системе Fuji.

## Аутентификация

API поддерживает два типа аутентификации:

### Для внешних систем (API-ключ)
```
x-api-key: ваш_api_ключ
```

### Для фронтенда и админки (JWT токен)
```
Authorization: Bearer ваш_jwt_токен
```

API-ключ можно получить у администратора системы.

## Формат данных

Все данные передаются и принимаются в формате JSON.

### Структура объекта зоны (БД)

```json
{
  "id": 1,
  "city_id": "a85360f2-55a8-47cc-8a79-1eb88a40c4f0",
  "zone_id": "zone-1",
  "terminal_id": "terminal-123",
  "address": "Самара, Георгия Димитрова, 131",
  "zoneName": "зона 10.1",
  "preparationTime": 30,
  "deliveryTime": 45,
  "zone_data": {
    "coords": [
      {"lat": 53.27958827953765, "lng": 50.275041898395095},
      {"lat": 53.28019768939508, "lng": 50.27429892649797}
    ],
    "fillColor": "#c8c8b4",
    "zoneId": "zone-1",
    "address": "Самара, Георгия Димитрова, 131"
  },
  "created_at": "2024-01-15T10:00:00.000Z",
  "updated_at": "2024-01-15T10:30:00.000Z"
}
```

### Структура объекта зоны (для фронтенда)

```json
{
  "coords": [
    {"lat": 53.27958827953765, "lng": 50.275041898395095},
    {"lat": 53.28019768939508, "lng": 50.27429892649797}
  ],
  "fillColor": "#c8c8b4",
  "zoneId": "zone-1",
  "zoneName": "зона 10.1",
  "address": "Самара, Георгия Димитрова, 131",
  "cityId": "a85360f2-55a8-47cc-8a79-1eb88a40c4f0",
  "terminalId": "terminal-123",
  "preparationTime": 60,
  "deliveryTime": 30
}
```

- `coords` (обязательно) - массив координат полигона, минимум 3 точки
- `fillColor` (обязательно) - цвет заливки полигона в формате HEX
- `zoneId` (обязательно) - уникальный идентификатор зоны
- `zoneName` (опционально) - название зоны в формате "зона X.Y"
- `address` (обязательно) - адрес зоны доставки
- `cityId` - ID города (UUID, 36 символов)
- `terminalId` - ID терминала доставки
- `zone_id` - UUID зоны (36 символов, альтернативное поле для zoneId)
- `city_id` - UUID города (36 символов)
- `terminal_id` - UUID терминала (36 символов)
- `deliveryId` - ID товара доставки из iiko (36 символов)
- `preparationTime` - время приготовления в минутах (по умолчанию 60)
- `deliveryTime` - время доставки в минутах (по умолчанию 30)

## Ошибки

API может возвращать следующие ошибки:

- `400 Bad Request` - ошибка валидации данных
- `401 Unauthorized` - отсутствует API-ключ или JWT токен
- `403 Forbidden` - недействительный API-ключ или недостаточно прав
- `404 Not Found` - запрашиваемый ресурс не найден
- `409 Conflict` - конфликт при создании ресурса
- `500 Internal Server Error` - внутренняя ошибка сервера

## Валидация данных

### Координаты
- Минимум 3 точки в массиве `coords`
- Широта (`lat`) от -90 до 90
- Долгота (`lng`) от -180 до 180

### Цвет заливки
- Формат HEX: `#RRGGBB` или `#RGB`
- Пример: `#c8c8b4`, `#fff`

### Идентификаторы
- `cityId` - UUID (36 символов)
- `zoneId` - строка минимум 1 символ
- `terminalId` - строка минимум 1 символ

## Кеширование

API использует внутреннее кеширование зон на 5 минут для оптимизации производительности. При изменении зон кеш автоматически очищается.

## Уведомления

При изменении зон доставки система автоматически:
- Обновляет ревизию данных
- Отправляет WebSocket уведомления подключенным клиентам
- Обновляет файлы зон для обратной совместимости

## Важные изменения

### Обновление системы терминалов
- **Терминалы теперь хранятся в базе данных** вместо статических файлов
- **Эндпоинт `/terminals`** возвращает данные из БД
- **Все терминалы имеют статус активности** (isDisabled: false/true)
- **Добавлены новые поля** для интеграции с iiko и платежными системами

### Новые возможности
- **Эндпоинт `/delivery/:zoneId`** - получение стоимости доставки по зоне
- **Эндпоинт `/all`** - получение всех зон из базы данных
- **Улучшенная валидация** данных зон
- **Кеширование** для повышения производительности

### Обратная совместимость
- Все существующие эндпоинты продолжают работать
- Формат ответов не изменился
- API-ключи остались прежними
