# API зон доставки - Документация для внешних систем

Документация описывает API для управления зонами доставки системы Fuji для внешних разработчиков.

## Аутентификация

Все запросы требуют API-ключ в заголовке:

```
x-api-key: ваш_api_ключ
```

API-ключ предоставляется администратором системы.

## Базовый URL

```
https://api.fuji.ru/api/v1/map/zones
```

В данный момент запросы должны быть к тестовому серверу:
```
https://fuji-web-api-v3.fuji.ru/api/v1/map/zones
```

## Формат данных

Все данные передаются в формате JSON. Кодировка UTF-8.

### Структура зоны доставки

```json
{
  "coords": [
    {"lat": 53.27958827953765, "lng": 50.275041898395095},
    {"lat": 53.28019768939508, "lng": 50.27429892649797},
    {"lat": 53.28034362572923, "lng": 50.27467711796907}
  ],
  "fillColor": "#c8c8b4",
  "zoneId": "zone-1",
  "zoneName": "зона 10.1",
  "address": "Самара, Георгия Димитрова, 131",
  "preparationTime": 60,
  "deliveryTime": 30
}
```

**Обязательные поля:**
- `coords` - массив координат полигона (минимум 3 точки)
- `fillColor` - цвет зоны в формате HEX (#RRGGBB или #RGB)
- `zoneId` - уникальный идентификатор зоны
- `address` - адрес зоны доставки

**Опциональные поля:**
- `zoneName` - название зоны в формате "зона X.Y" (автоматически генерируется если не указано)
- `city_id` - UUID города (36 символов)
- `terminal_id` - UUID терминала (36 символов)
- `deliveryId` - ID товара доставки из iiko (36 символов)
- `preparationTime` - время приготовления в минутах (по умолчанию 60)
- `deliveryTime` - время доставки в минутах (по умолчанию 30)

### Обновление зоны для терминала

```
PUT /:cityId/terminal/:terminalId
```

Создает новую зону или обновляет существующую для терминала.

**Параметры:**
- `cityId` - ID города (UUID)
- `terminalId` - ID терминала

**Тело запроса:** (аналогично POST)

**Ответ:**
- `201 Created` - если зона создана впервые
- `200 OK` - если зона обновлена

### Удаление зоны

```
DELETE /:zoneId
```

**Параметры:**
- `zoneId` - ID зоны для удаления

**Пример запроса:**
```bash
curl -X DELETE "https://fuji-web-api-v3.fuji.ru/api/v1/map/zones/af03ac3e-dee5-f415-0181-0524010d81aa" \
  -H "x-api-key: ваш_api_ключ"
```

**Ответ (204 No Content):**
Тело ответа отсутствует при успешном удалении.

## Валидация данных

### Координаты
- Минимум 3 точки в массиве `coords`
- Широта (`lat`): от -90 до 90
- Долгота (`lng`): от -180 до 180
- Координаты должны быть числами
- Точки должны образовывать замкнутый полигон

### Цвет заливки
- Формат HEX: `#RRGGBB` или `#RGB`
- Примеры: `#c8c8b4`, `#fff`, `#ff0000`

### Идентификаторы
- `cityId` - UUID (36 символов)
- `zoneId` - UUID (36 символов)
- `terminalId` - UUID (36 символов)

## Коды ошибок

- `400 Bad Request` - некорректные данные запроса
- `401 Unauthorized` - отсутствует или неверный API-ключ
- `404 Not Found` - ресурс не найден
- `409 Conflict` - конфликт при создании (зона уже существует)
- `500 Internal Server Error` - внутренняя ошибка сервера

## Примеры ошибок

**400 Bad Request:**
```json
{
  "error": "Bad Request",
  "message": "coords должен быть массивом с минимум 3 точками"

```

**404 Not Found:**
```json

  "message": "Зона с идентификатором af03ac3e-dee5-f415-0181-0524010d81aa не найдена"
```

```json
{
  "error": "Conflict",
```
